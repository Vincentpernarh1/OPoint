# Vpena OnPoint - Enterprise Time Tracking & HR Management System

A comprehensive, multi-tenant HR management and time-tracking application built with modern web technologies, featuring offline capabilities, mobile support, and real-time push notifications.

## ğŸ—ï¸ Architecture Overview

### Frontend Stack
- **React 19** with TypeScript for type-safe component development
- **Vite** for fast development and optimized production builds
- **React Router DOM** for client-side routing and navigation
- **Tailwind CSS** for utility-first styling
- **Capacitor** for cross-platform mobile app deployment (iOS/Android)
- **IndexedDB** (via `idb` library) for offline data storage and synchronization

### Backend Stack
- **Node.js** with Express.js framework
- **Supabase** (PostgreSQL) for database with Row Level Security (RLS)
- **JWT-based authentication** with secure cookie management
- **Web Push API** for mobile push notifications
- **bcrypt** for password hashing
- **Nodemailer** for email services
- **Multer** for file uploads

## ğŸš€ Quick Start

### Prerequisites
- Node.js 18+
- Supabase account and project
- Mobile development environment (for Capacitor builds)

### Installation

1. **Clone and Install Dependencies**
   ```bash
   git clone <repository-url>
   cd vpena-onpoint
   npm install
   ```

2. **Environment Configuration**
   ```bash
   # Create .env file with required variables
   SUPABASE_URL=your_supabase_project_url
   SUPABASE_ANON_KEY=your_supabase_anon_key
   SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
   JWT_SECRET=your_jwt_secret
   VAPID_PUBLIC_KEY=your_vapid_public_key
   VAPID_PRIVATE_KEY=your_vapid_private_key
   ```

3. **Database Setup**
   - Execute SQL migrations in order:
     - `create_tenant_tables.sql`
     - `create_time_adjustments_table.sql`
     - `create_leave_balances_table.sql`
     - `create_expense_claims_table.sql`
     - `create_announcements_tables.sql`
     - `create_mobile_push_tokens_table.sql`
     - `create_push_subscriptions_table.sql`
     - `migration_company_isolation.sql`
     - Performance indexes: `add_performance_indexes.sql`

4. **Development Server**
   ```bash
   npm run dev
   ```

5. **Backend Server**
   ```bash
   npm start
   ```

### Mobile Build

1. **iOS/Android Setup**
   ```bash
   npx cap add ios
   npx cap add android
   ```

2. **Build and Sync**
   ```bash
   npm run build
   npx cap sync
   ```

3. **Open in IDE**
   ```bash
   npx cap open ios
   npx cap open android
   ```

## ğŸ“ Project Structure

### Core Application Files

- `index.tsx` - Application entry point with React 19 rendering
- `server.js` - Express.js backend server with API endpoints
- `package.json` - Node.js dependencies and scripts configuration

- `services/authService.ts` - Authentication and session management
- `services/database.ts` - Supabase client and database operations
- `services/api.ts` - REST API client for backend communication
- `services/offlineStorage.ts` - IndexedDB wrapper for offline functionality
- `services/pushService.ts` - Web Push notification management

### Database Layer
- `DATABASE_SETUP.md` - Complete database schema documentation
- Multiple SQL migration files for table creation and updates
- Row Level Security policies for multi-tenant data isolation

### Mobile Configuration
- `capacitor.config.ts` - Capacitor configuration for mobile builds
- `vite.config.ts` - Vite configuration with PWA plugins

### Components Architecture
- Lazy-loaded React components for optimal performance
